## 1.1 원본 파일 받는 방법

`ai-server/data/regulations/`는 `.gitignore` 처리돼 있어 `git pull`로는 원본 PDF·추출 텍스트가 안 딸려온다.

**4건 전부 law.go.kr 공식 원문**(법률·시행령·시행규칙 3건 + 행정규칙 1건)이며, law.go.kr은 자동 다운로드가 막혀 있어 **전부 수동 확보만 가능**하다. 애초 시행령·시행규칙 2건은 assi.or.kr 협회 미러를 자동 다운로드해 썼으나 버전이 불확실해 팀원이 확보한 공식 원문으로 교체했다(§1 상태 칸 참고) — 그래서 curl 재현 명령은 더 이상 유효하지 않다.

이은석이 4건 전부 Google Drive 업로드 예정(§1 표의 `TODO` 자리에 링크 채울 예정). 추출 텍스트가 필요하면 원본 PDF를 받은 뒤 `pdftotext -layout {파일} {출력}.txt`로 직접 생성(macOS: `brew install poppler`).

## 2. 청킹 실험 결과 — #1 시설물안전법

기존 `ai/core/chunking.py`의 `split_regulation_text()`를 신규 구현 없이 그대로 사용(오영석님 HAJA-20④ 구현·검증 재사용).

- **추출**: `pdftotext -layout` (텍스트 PDF 23p 원본)
- **청크 수**: 138개, 평균 327자, 최대 499자, 최소 10자
- **제N조 경계 시작 비율**: 98/138 (71%)
- **원인 메모**: law.go.kr PDF는 본문 앞에 조문 제목만 나열한 **목차(찾아보기) 섹션**이 포함되어 있어, 이 구간도 "제N조"로 매칭되면서 내용 없는 짧은 청크(제목만, 10~20자)가 다수 발생 — 오영석님이 검증한 국토부고시 원문(목차 없는 순수 본문, 74/89=83%)보다 비율이 낮게 나온 원인으로 추정. **본문 파싱 전 목차 구간 스킵 로직 추가는 이번 이슈 범위 밖**(청킹 함수 자체는 정상 동작 확인됨) — 필요 시 후속 이슈로 분리.
- **참고**: 여기서 비교 대상으로 든 "오영석님이 검증한 국토부고시 원문(72KB, 제1~56조)"은 그분이 HAJA-20④ 커밋 당시 로컬에서 쓴 임시 테스트 파일로, **레포에 포함돼 있지 않아 재사용·재검증이 불가능**하다. §1의 문서 #4(국토교통부고시 제2025-470호)는 이번 세션에서 팀원을 통해 별도로 확보한 **다른 문서**이며, 그 문서의 완료 판정은 오영석님 검증치가 아니라 §3에 기록된 이번 세션 자체 청킹 실험 결과에 근거한다.

## 3. 청킹 실험 결과 — #2~4

| 문서 | 원문 길이 | 청크 수 | 평균 길이 | 제N조 시작 비율 | 사용 함수 |
|---|---|---|---|---|---|
| 시행령(제36220호, 공식본) | 29,548자 | 83개 | 368자 | 71/83 (86%) | `split_regulation_text` |
| 시행규칙(제1553호, 공식본) | 23,179자 | 62개 | 390자 | 56/62 (90%) | `split_regulation_text` |
| 실시 지침 고시(제2025-470호, 185p) | 217,669자 | 622개 | 346자 | 146/622 (23%) | `split_regulation_text` |

본법(71%)보다 시행령·시행규칙 비율이 높은 건 목차 섹션이 짧거나 없어서로 추정 — §2의 목차 혼입 가설과 일치.

실시 지침 고시는 23%로 4건 중 가장 낮다 — 185p 중 상당 분량이 서식·별지·평가표 등 표 형태 부록으로 추정되며, 이런 구간은 "제N조" 패턴이 없어 `split_regulation_text`가 일반 텍스트처럼 잘게 쪼갠다. 부록 구간을 분리해 별도 처리할지는 실제 검색 품질 확인 후 판단(§4).
