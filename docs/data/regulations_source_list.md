## 2. 청킹 실험 결과 — #1 시설물안전법

기존 `ai/core/chunking.py`의 `split_regulation_text()`를 신규 구현 없이 그대로 사용(오영석님 HAJA-20④ 구현·검증 재사용).

- **추출**: `pdftotext -layout` (텍스트 PDF 23p 원본)
- **청크 수**: 138개, 평균 327자, 최대 499자, 최소 10자
- **제N조 경계 시작 비율**: 98/138 (71%)
- **원인 메모**: law.go.kr PDF는 본문 앞에 조문 제목만 나열한 **목차(찾아보기) 섹션**이 포함되어 있어, 이 구간도 "제N조"로 매칭되면서 내용 없는 짧은 청크(제목만, 10~20자)가 다수 발생 — 오영석님이 검증한 국토부고시 원문(목차 없는 순수 본문, 74/89=83%)보다 비율이 낮게 나온 원인으로 추정. **본문 파싱 전 목차 구간 스킵 로직 추가는 이번 이슈 범위 밖**(청킹 함수 자체는 정상 동작 확인됨) — 필요 시 후속 이슈로 분리.
