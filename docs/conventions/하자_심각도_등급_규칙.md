## 1. 목적·범위

- 탐지된 하자 각각에 대해 **심각도를 A~E 5등급으로 산정**하는 규칙을 정의한다.
- 대상 클래스 3종(확정): **균열 · 박리·박락 · 철근 노출**.
- **규칙기반 1차**(크기·유형 기준). 산정 결과는 점검자가 검수 단계에서 **수동 조정 가능**(PRD FR-4, `defect_revisions` append-only 기록).
- 시설물 안전등급(A~E) 관행을 준용 — **A=우수(경미), E=불량(심각)**.

---

## 2. 등급 정의 (A=우수 ~ E=불량)

**등급 점수 `g ∈ [0, 1]`** — 높을수록 양호. 하자가 심할수록 낮은 점수를 받아 낮은 등급(E)으로 간다.
`g = 1 − s` (s = 심각도 원점수, §3). **0.2 단위** 버킷.

| 등급 | 점수 `g` 범위 | 의미 | 안전등급 준용 |
|---|---|---|---|
| **A** | 0.8 ≤ g ≤ 1.0 | 경미 | 우수 |
| **B** | 0.6 ≤ g < 0.8 | 양호 | 양호 |
| **C** | 0.4 ≤ g < 0.6 | 보통 | 보통 |
| **D** | 0.2 ≤ g < 0.4 | 주의 | 미흡 |
| **E** | 0.0 ≤ g < 0.2 | 심각 | 불량 |

**경계 처리(off-by-one 방지)**: `g ≥ 0.8 → A`, `0.6 ≤ g < 0.8 → B`, `0.4 ≤ g < 0.6 → C`, `0.2 ≤ g < 0.4 → D`, `g < 0.2 → E` (상단 닫힘 A만 1.0 포함, 나머지 `[lo, hi)` 반열림).

---

## 3. 규칙 1 — 심각도 원점수 `s` 와 임계값 (크기 · 유형)

심각도 원점수 **`s ∈ [0, 1]`** (높을수록 심각) 을 **크기 프록시 × 유형 가중**으로 산출한다.

### 3.1 크기 프록시 (1차)
- 정량 실측(균열 폭·길이 mm)은 PRD **P1**(픽셀→실측 환산). **1차 규칙기반에서는 YOLOv8-seg 산출물의 마스크 면적 비율을 프록시**로 사용한다.
- `area_ratio = (하자 마스크 픽셀 면적) / (이미지 전체 픽셀 면적)` — 0~1.

### 3.2 유형별 산정 (초안 — 튜닝 대상)

| 유형 | 크기 프록시 | s 산정(1차 초안) | 비고 |
|---|---|---|---|
| 균열(crack) | 마스크 길이/면적 비율 | area_ratio 구간 매핑(아래) | P1에서 폭·길이 정량으로 대체 |
| 박리·박락(spalling) | 마스크 면적 비율 | area_ratio 구간 매핑 | 면적형 |
| 철근 노출(rebar) | 존재 + 면적 비율 | **기본 하한 s₀=0.6** + area 가중 | 존재 자체가 고위험 → 최소 D 이하로 시작 |

**area_ratio → s 구간(공통 초안, %기준)**:

| area_ratio | s | (철근노출 보정 후) |
|---|---|---|
| < 1% | 0.1 | max(0.1, 0.6) = 0.6 |
| 1 ~ 3% | 0.3 | 0.6 |
| 3 ~ 7% | 0.5 | 0.6 |
| 7 ~ 15% | 0.7 | 0.7 |
| ≥ 15% | 0.9 | 0.9 |

> 위 임계값(%·구간·철근 하한 0.6)은 **1차 가정치**다. 데이터셋(약 161,000장) 라벨 분포 확인 후 실측 기준으로 재조정한다. 한 이미지에 하자 다수 시 등급은 **하자별로 개별 산정**(집계 등급이 필요하면 최저 등급 = 최악 하자 기준).

## 4. 규칙 2 — 산정 위치 · 구현 (109 범위 밖, 후속 dev)

- **산정 위치 = FastAPI 탐지 후처리**(권장). YOLOv8-seg 추론 결과(class, mask, area) → `s` → `g` → grade 계산 → **추론 결과 payload에 `grade` 포함**해 반환.
  - 근거: `ai-server/ai/chains/defect_explain_chain.py` 가 이미 `severity_grade: str` 를 **입력으로 기대** → 등급은 상위(탐지 파이프라인)에서 계산해 넘기는 구조가 이미 전제됨.
- **임계값 저장**: 1차 = ai-server **상수 모듈 하드코딩**(예: `ai/core/grading.py` 상수) → 후속으로 관리자 **"하자 유형·등급 기준 관리"(dev-11-03, FR-8)** 에서 DB화·설정형 전환.
- **저장 스키마**: `defects.grade` (A~E enum) 컬럼 — 하자 엔티티(하자 관리, 유병현) 생성 시 반영. `defects`에 원점수(`severity_score`)도 함께 저장하면 재계산·감사에 유리.
- **수동 조정**: 점검자 검수 시 등급 변경 → `defect_revisions` append-only 기록(FR-4 감사 추적).
