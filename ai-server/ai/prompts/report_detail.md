아래는 확정된 하자 목록 {defect_count}건입니다. **목록의 각 하자마다 정확히 1개씩** 상세 항목을 생성하세요(추가·누락·병합 금지 — 입력 개수와 출력 items 개수가 반드시 일치해야 합니다).

## 확정 하자 목록

{defects_list_text}

## 출력 (items: 위 목록과 1:1 대응하는 배열, 총 {defect_count}개)

각 항목마다:
1. **defect_type**: 하자 유형 (목록의 값 그대로)
2. **location**: 발생 위치 (목록의 값 그대로)
3. **severity_grade**: 하자 등급 A~E (목록의 값 그대로)
4. **description**: 상태 서술 (목록의 설명을 근거로 구체화, 없는 사실 창작 금지)
5. **cause**: 추정 원인 (하자 유형·등급·위치를 근거로 한 원인, 2~3문장)

목록에 없는 하자를 추가하거나, 목록의 항목을 누락·병합하지 마세요.
